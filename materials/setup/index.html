Setup

To follow along with the course exercises, you’ll need R, Quarto, a code editor that supports R, and a few R packages.

While the workshop isn’t centered around Quarto, the exercises and examples will all be based on Quarto documents so that text and code and output can all be included together, making a nicer resource for future you.

Recommended setup options

There are two ways to get set up for the workshop:

  1. Local installation
  2. Posit Cloud

While Posit Cloud lets you run RStudio directly in your browser and makes it so you don’t need to install anything on your computer (convenient!), I highly recommend installing R and RStudio/Positron on your computer. The Bayesian models we’ll run won’t run well on Posit Cloud, which by default only has once CPU core and limited RAM.

Install {CmdStanR}

We’ll use Stan through {brms} to run the Bayesian models. You should have already installed {brms} in the install.packages() command above.

NoteRaw Stan

If you’ve never worked with Stan, don’t worry! You won’t have to write a single line of Stan code in this workshop. {brms} translates familiar R model syntax into Stan code, runs the MCMC sampler, and bring the results back into R, all without you needing to see what’s happening. It’s magical.

Stan has two R-based interfaces for communicating between the two programs: RStan and CmdStanR. {brms} works with both backends. I recommend installing CmdStan. CmdStan is newer and has more modern features (like speeding up sampling with multi-threading), and it works nicely with the more complex multinomial models we’ll be running.

The {CmdStanR} package helps you install CmdStan on your computer and provides the actual interface to bridge Stan and R. It is not on CRAN because it requires other binary software for connecting to Stan, so you have to install it a little differently:

  1. Install the {CmdStanR} package. Use this command in R to install the package from Stan’s R-Universe repository:

    install.packages("cmdstanr", repos = c('https://stan-dev.r-universe.dev', getOption("repos")))

  2. Install a C++ toolchain. Stan models need to be compiled to binary programs to run. See here for complete instructions for installing the required tools. In short:

    • On Windows, download and install Rtools45: https://cran.r-project.org/bin/windows/Rtools/

    • On macOS, install Xcode’s command line tools by running this in a terminal:

      xcode-select --install

    • On Linux, install g++ and make. make is typically pre-installed, and g++ is generally also installed. If not, run something like this to install them (this assumes a Debian-based distribution like Ubuntu; adapt the command as necessary for your distribution):

      sudo apt install g++ make

  3. Make sure the C++ toolchain is working. In R, run this in your console to make sure everything’s set up correctly:

    cmdstanr::check_cmdstan_toolchain()

  4. Install CmdStan. Finally, you need to compile and install CmdStan. You can do that with {cmdstanr} in R. Run this in your console:

    cmdstanr::install_cmdstan()

  5. Make sure everything works. Run this in R to make sure everything’s working correctly:

    library(brms)

model <- brm(
  bf(mpg ~ disp + am),
  data = mtcars,
  family = gaussian(),
  backend = "cmdstan"
)
